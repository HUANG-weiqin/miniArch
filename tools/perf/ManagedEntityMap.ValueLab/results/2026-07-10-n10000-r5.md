## Notes

- **RawDenseUnsafeMap is a performance upper bound / incorrect example.**
  It does NOT call `world.IsAlive(entity)` for liveness checks.
  It can return stale data for destroyed entities whose slot was reused.
  It serves as an unattainable performance ceiling for correct implementations.

- **Managed sidecar is host-local, non-deterministic, and non-thread-safe.**
  It does not participate in lockstep, FrameDelta, Snapshot, Checksum, or Replay.
  `world.IsAlive(entity)` is the only liveness oracle; sidecar versions only represent
  slot binding state.

- Clear and TrimExcess are measured as single operations (1 op per call).
  All other operations count per-entity invocations as ops.
- Alloc/op may be inflated for Clear/TrimExcess since they represent batch-level
  allocation for the single call, not divided by entity count.
- Ret KB = retained memory after measurement runs (GC.GetTotalMemory delta from baseline).
